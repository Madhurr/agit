Write a TypeScript module for reading agit git notes via the GitHub REST API.

File: github-app/src/notes-reader.ts

This module reads agit commit metadata (stored as git notes in refs/notes/agit) from a GitHub repository using ONLY the GitHub REST API — no git clone needed.

How agit git notes work:
- Notes are stored in the ref refs/notes/agit
- Each note is a blob whose path in the notes tree is the commit SHA it annotates
- The blob content is a JSON string matching the CommitNote schema

Function to implement:

```typescript
export async function readNote(
  octokit: Octokit,
  owner: string,
  repo: string,
  commitSha: string
): Promise<CommitNote | null>
```

Algorithm:
1. Call GET /repos/{owner}/{repo}/git/refs/notes/agit to get the notes ref SHA
   - If 404, return null (no notes in this repo)
2. Call GET /repos/{owner}/{repo}/git/trees/{sha}?recursive=1 to list all blobs
   - tree entries are objects with "path" (commit SHA) and "sha" (blob SHA)
3. Find the tree entry whose path equals commitSha (full 40-char SHA)
   - If not found, return null (no note for this commit)
4. Call GET /repos/{owner}/{repo}/git/blobs/{sha} to get the blob
   - blob.content is base64 encoded, blob.encoding is "base64"
5. Decode base64 content: Buffer.from(blob.content, 'base64').toString('utf8')
6. Parse JSON and return as CommitNote

Import: import { Octokit } from '@octokit/core';

Error handling:
- Catch HTTP 404 errors and return null (don't throw)
- Let other errors propagate (network issues, auth failures)
- Use try/catch around the entire function body
- Check status codes: if response status is 404, return null
